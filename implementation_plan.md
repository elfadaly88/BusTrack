# خطة عمل منصة تتبع الحافلات متعددة المؤسسات (BusTrack SaaS Multi-Tenant)
# BusTrack SaaS Multi-Tenant Implementation Plan

هذه وثيقة خطة العمل المحدثة لتحويل نظام تتبع الحافلات إلى منصة **SaaS متعددة المؤسسات (Multi-Tenant)** تدار بواسطة مسؤول المنصة العام (**Super Admin**)، وتسمح لكل مؤسسة (مدرسة، جامعة، شركة) بإدارة مستخدميها ومساراتها بشكل معزول تماماً.

* **شاشة لوحة تحكم المسؤول العام (Super Admin Dashboard) [NEW]**:
  * عرض إحصائيات إجمالية لجميع المشتركين (إجمالي المؤسسات، إجمالي الرحلات النشطة على مستوى المنصة).
  * جدول لإدارة المؤسسات (تفعيل/تعطيل الاشتراك، إنشاء جهة جديدة).

## 📋 توزيع الأدوار وتفاصيل شاشات المستخدمين الجديدة

### 1️⃣ المسؤول العام للمنصة (Super Admin) - [NEW]
* **لوحة التحكم السحابية (Super Admin HUD)**:
  * شاشة عرض إحصائيات عامة للمنصة.
  * إدارة المستأجرين (Tenants Management):
    * إضافة مؤسسة جديدة (الاسم، المعرف النصي `slug`).
    * تعيين حساب الأدمن الأول للمؤسسة (الاسم، البريد الإلكتروني، كلمة المرور).
    * إيقاف مؤقت أو تفعيل المؤسسة (يؤدي إيقاف المؤسسة لحظر جميع مستخدميها من تسجيل الدخول فوراً).

### 2️⃣ مسؤول المؤسسة (Tenant Admin) - [UPDATED]
* يدير فقط البيانات التابعة لـ `tenant_id` الخاص به:
  * إضافة حافلات وسائقي وركاب المؤسسة.
  * تصميم المسارات ورسم الخرائط للخطوط التابعة لمؤسسته فقط.
  * جدولة وإطلاق الرحلات.

## 🗄️ معمارية قاعدة البيانات وتوزيع المستأجرين (Database Schema)

نقوم بتحديث قواعد البيانات بالكامل لتدعم تعدد الجهات (Multi-tenancy):

### الجداول المحدثة والجديدة:
1. **المؤسسات (`tenants`) [NEW]**:
   * `id` INTEGER PRIMARY KEY AUTOINCREMENT
   * `name` TEXT NOT NULL
   * `slug` TEXT UNIQUE NOT NULL (معرّف الرابط، مثل `cairo-uni`)
   * `status` TEXT CHECK(status IN ('active', 'suspended')) DEFAULT 'active'
   * `created_at` TEXT NOT NULL

2. **المستخدمين (`users`) [UPDATED]**:
   * إضافة حقل `tenant_id` INTEGER (يسمح بـ NULL للمسؤول العام فقط).
   * إضافة دور جديد `super_admin` في فحص القيد.

3. **الحافلات (`buses`) [UPDATED]**:
   * إضافة حقل `tenant_id` INTEGER NOT NULL.
   * تعديل قيد التفرد ليكون على مستوى المؤسسة: `UNIQUE(bus_number, tenant_id)`.

4. **الخطوط (`lines`) [UPDATED]**:
   * إضافة حقل `tenant_id` INTEGER NOT NULL.

5. **الرحلات (`trips`) [UPDATED]**:
   * إضافة حقل `tenant_id` INTEGER NOT NULL.
